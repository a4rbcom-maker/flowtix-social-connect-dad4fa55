سأصلح مشكلة الاختبار والحفظ على صفحة ربط فيسبوك بحيث لا تظهر رسالة “رد الخادم غير مكتمل” مع التوكن الصحيح.

الخطة:

1. توحيد شكل ردود دوال فيسبوك
- جعل `testFacebookToken` و `connectFacebook` يرجعان دائماً نفس الشكل الآمن:
  - `success`
  - `profile`
  - `granted`
  - `declined`
  - `error: null`
- عدم الاعتماد على وجود `name` من فيسبوك؛ يكفي وجود `id`، وإذا الاسم غير موجود نعرض اسم بديل آمن.

2. جعل أخطاء فيسبوك لا تكسر استدعاء الخادم
- بدلاً من أن ترمي دوال الاختبار/الحفظ خطأ خام من Facebook API أو قاعدة البيانات، سترجع نتيجة منظمة فيها `error`.
- هذا يمنع TanStack server functions من إرجاع رد غير قابل للتفسير للواجهة.

3. إصلاح واجهة الاختبار والحفظ
- تحديث `normalizeAuthResponse` ليقبل كل الأشكال المحتملة للرد، بما فيها الرد المغلف أو الرد المنظم الجديد.
- إذا جاء `error` من الخادم، تعرض الواجهة رسالة دقيقة: توكن منتهي، صلاحيات ناقصة، فشل حفظ، أو مشكلة شبكة.
- زر “اختبار التوكن” وزر “ربط الحساب” سيستخدمان نفس منطق المعالجة بدون افتراض وجود `profile` أو `granted`.

4. إضافة تشخيص واضح داخل وضع Debug
- عند تفعيل Debug، نسجل خطوة الاختبار/الحفظ وشكل الرد المختصر بدون كشف التوكن.
- الهدف أن تظهر المشكلة القادمة بوضوح بدلاً من رسالة عامة “رد الخادم غير مكتمل”.

5. التحقق
- تشغيل فحص الملفات المعدلة للتأكد من عدم وجود أخطاء TypeScript/ESLint.
- التأكد أن مسار الصفحة لا يعود يستخدم `.profile` أو `.includes` على بيانات غير مضمونة.
