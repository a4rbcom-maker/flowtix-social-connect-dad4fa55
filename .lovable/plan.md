## التحقق من deploy.yml

**ملاحظة:** لا أقدر أشغّل GitHub Actions من هنا — التشغيل لازم يتم من واجهة GitHub أو عبر `gh` CLI على جهازك. لكن قدرت أعمل فحص ستاتيكي محلي وهو اللي بيقيس نفس الشرط (طول التعبيرات + صحة الـ workflow).

### نتائج الفحص المحلي

| الفحص | النتيجة |
|---|---|
| صحة YAML (`yaml.safe_load`) | ✅ صالح |
| عدد الأسطر | 1061 |
| أطول تعبير `${{ ... }}` | **54 حرف فقط** (السطر 326: `${{ inputs.integrity_rollback_dry_run && '1' \|\| '0' }}`) |
| الحد المسموح به من GitHub | 21000 حرف |
| الهامش المتبقي | ~99.7% |

أطول 10 تعابير كلها تحت 60 حرف — لا يوجد أي تجاوز ولا أي تعبير قريب من الحد.

### الخطوة المطلوبة منك (تشغيل فعلي على GitHub)

شغّل الـ workflow عشان الـ runner يأكد نفس النتيجة:

**من الواجهة:**
Actions → Deploy workflow → **Run workflow** → بدون تفعيل الـ dry-run (عشان تشوف التحقق على نشر حقيقي) أو معاه (للتجربة الآمنة).

**من الـ CLI:**
```bash
gh workflow run deploy.yml
gh run watch
```

### الخطة لو فيه أي إشارة لفشل بعد التشغيل

1. أجيب الـ logs (`gh run view --log-failed`) أو سكرين شوت منك.
2. أحدد السطر/الخطوة بالضبط.
3. أعدّل الجزء المعني فقط بدون لمس باقي الـ workflow.

لو الـ logs طلعت نظيفة، الموضوع متأكد منه ومش محتاج تغيير.
